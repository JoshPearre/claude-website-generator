- **Source:** VibeCodeComponents

- **Type:** text-prompt
- **Intensity:** 3
- **Tags:** form,voice,speech-to-text,web-speech-api
- **Best for tiers:** 3,4
- **Prompt:**
  > Create a voice typing component using the Web Speech API:
  >
  > UI Requirements:
  > - Microphone button that toggles recording
  > - Visual feedback when recording (pulsing animation)
  > - Stop button to end recording
  > - Show interim/partial transcription in real-time
  > - Handle browser compatibility gracefully (show message if unsupported)
  >
  > Transcription Behavior (Critical):
  > - Accumulate all transcript segments during the session — don't send each segment individually
  > - Only finalize the complete, accumulated transcript when the user clicks Stop

  > - Use refs (not just state) for tracking recording status to avoid stale closure issues in event handlers
  > - Show real-time transcription in the text box
  > - Add a short delay (500-1000ms) after stopping to allow final results to process before flushing the transcript
